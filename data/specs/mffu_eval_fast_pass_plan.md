## 5. ★推奨案とその理由

### 推奨: **案B 中庸 (MES 3-5枚・15-22日通過狙い)**

#### 選定理由
1. **Pass確率 65-72%** が3案中最高リスクリワード
2. Chronos ORB 実績 (VIX>20 でSharpe 3.04) を素直に活用
3. MLL -$2,000 を 8日で使い切るリスクが少ない (-$250/日×8=$2000)
4. Consistency 50% を自然にクリア (単日 $250-700 分散)
5. 15-22日 = 3-4.5週 = 退職タイムライン (5/31) に十分間に合う
6. Failed時の再挑戦 $107 × 2 = $214 のバッファあり
7. VIX<20 期は発注停止で無損失保持可能

#### 案Aを推奨しない理由
- 8枚サイジングでMLL 40%/日消費 → 1回のVIX急騰逆指値ヒットで事故
- Pass確率 45-55% は「半分以上失敗」と同義・退職タイムライン圧迫時に裏目
- 「最速で進める誘惑」は感情バイアス (ゆうさくさん目標早期化規律) だが**期待通過日数と失敗率の積で評価すると B > A**
  - A: 11日×0.5 + 失敗後再挑戦(11日)×0.5 = 16.5日期待
  - B: 18日×0.68 + 再挑戦(18日)×0.32 = 18日期待
  - **差は1.5日のみ。失敗ダメージは案Aが大きい**

#### 案Cを推奨しない理由
- Pass確率は高いが 28-35日 × 試行回数で時間コスト過大
- 退職 5/31 にEval通過できないと Funded 移行遅延
- Phase1 Sim-Funded $10K到達 (Live移行) も遠のく
- 1-2枚は MES $0.50/pt で稼ぎが薄く Consistency 違反リスクは低くなるが機会損失大

### 日次 Hard Limits
- **日次損失 $250 到達 → 当日 chronos 停止** (既存 halt_streak=3)
- **日次利益 $500 到達 → 当日 chronos 停止** (Consistency保護・新規実装要)
- **週次損失 $500 到達 → 当週停止** (MLL $2,000の25%)

### Consistency 50% 自動ガード (新規実装推奨)
```python
# chronos_bot.py に追加:
def check_consistency_guard(cumulative_pnl, daily_pnl_today, target=3000):
    if cumulative_pnl + daily_pnl_today <= 0:
        return True  # 累計赤字ならガード不要
    max_allowed_today = (cumulative_pnl + daily_pnl_today) * 0.50 - cumulative_pnl
    return daily_pnl_today <= max_allowed_today
```

| シグナル | 条件 | アクション |
|---|---|---|
| 🟡 警告 | MLL消費率 30% (-$600) | サイズ1枚減 (4→3) |
| 🟠 重大 | MLL消費率 50% (-$1,000) | **全停止1日・戦術再評価** |
| 🔴 緊急 | MLL消費率 75% (-$1,500) | **Eval撤退判断** (残り$500で再挑戦優位) |
| ⚠ Consistency | 単日利益 > 累計の45% | **その日は以降ノートレード** |
| ⚠ 進捗遅延 | Day10で $800未満 | **案C格下げ検討** |
| ⚠ 進捗超過 | Day2で $1,500超過 | **1-2日トレード停止** (Consistency違反回避) |

- Day 18 で profit < $1,500 (目標の50%未満) → **案B継続不可・撤退 ($500 MLL温存して新規購入)**

- 連敗原因分析 → VIXフィルタ調整 → 翌日再開
