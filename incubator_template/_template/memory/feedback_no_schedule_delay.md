# 先延ばし語彙禁止規律

## 禁句 TOP

- 「明日」「明朝」「明日中」「明日以降」
- 「後日」「後で」「後回し」「後ほど」
- 「来週」「週末に」「週明けに」
- 「次回」「一旦保留」「継続確認」
- 「本日は見送り」

## 原則

今日できないことは構造の問題。**今日できる最小単位に分解** すること。

## 例

```
✗ 明日 pytest を整備します
○ 今から 30 分で test_payment.py の 3 ケースを書く。残りは別タスク化

✗ 後日 redteam を回します
○ 今 redteam session を別 tab で起動・15 分で指摘 5 件出させる

✗ 週末にゆっくり考えます
○ 今 15 分で 3 案を箇条書きで出す。判断は本人に任せる
```

## 物理ガード

- `discipline_guard.sh`: PreToolUse/UserPromptSubmit で先延ばし語検出 + 3回目で hard block
- `deferral_language_guard.sh`: Stop hook で応答内の先延ばし語検出 + Pushover 通知

## 例外

- ユーザー本人の都合(休息・家族時間)で「明日対応」と user 自身が言う場合は OK
- ルール説明文脈で「『明日』禁句」と言及するのは OK(hook が文脈判定で skip)
