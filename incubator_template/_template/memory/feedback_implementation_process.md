# 実装前 7 ステップ規律

## 結論

実装に着手する前、以下 7 ステップを順守。途中省略は虚偽完了の元。

## 7 ステップ

1. **目的の言語化**: 何を達成したいか1文で
2. **公式仕様確認**: ライブラリ・API・コマンドの公式 doc を引く(`curl` or `gh` で fetch)
3. **既存実装調査**: 同等機能が既に存在しないか grep
4. **失敗パターン premortem**: この実装が失敗するなら何が原因か 3 つ挙げる
5. **テスト先行**: 期待挙動を pytest で書く
6. **実装**: テストを通す
7. **証跡 4 点セット**: grep + AST + pytest stdout + mutation/coverage

## アンチパターン

- 「とりあえず動かしてみる」 → 動いても何故動いたか分からず再現性なし
- 「テストは後で書く」 → 書かれない
- 「pytest 通った気がする」 → stdout なしで気のせい
- 「mutation score とか過剰」 → 0回でも 1 個だけ計測して 50% 切るかを見る

## 物理ガード

- `discipline_guard.sh`: 「2-3 日」「4-6 時間」「半日」等の桁違い見積を検知
- `false_claim_detector.sh`: 完了宣言時の pytest 証跡確認
